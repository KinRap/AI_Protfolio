3DSmart AI
Orchestrator

A 5-agent orchestration system that automates tender analysis in the renewable energy sector. Cuts analysis time from hours to under 6 minutes. Delivers a GO / CONDITIONAL / NO-GO executive recommendation.

The problem

Tender analysis in the renewable energy sector is slow, expensive, and error-prone. A single procurement dossier can run to hundreds of pages: notices, contract drafts, technical specifications, bill of quantities, opening information. A human expert needs 3–8 hours per tender to check compliance, flag risks, and produce a recommendation. Miss a penalty clause or a conflicting deadline and the consequences are contractual.

3DSmart needed this process automated — structured, auditable, fast, and accurate enough for management to act on.

Architecture — 5-agent pipeline

Each agent has a single responsibility, a defined personality, operating rules, and a structured JSON output that becomes the next agent's input. The inter-agent contracts are enforced at runtime — no agent proceeds without a complete upstream output.

1
DNPA — Data Normalisation & Preparation
Converts raw PDF/DOCX tender documents into structured, unambiguous JSON. Identifies conflicts, marks missing data, standardises formats. Temperature: 0.0 — precision only, no interpretation.
Output: normalised JSON + conflict log
↓
2
FEA — Fact Extraction Agent
Extracts 14 mandatory fact categories (submission deadlines, penalties, warranties, insurance requirements, evaluation criteria…). Every fact carries a source tag. More than 4 NO_DATA → pipeline halts with INPUT_INCOMPLETE.
Output: 14-category JSON with source tags
↓   parallel ↓
3
CRAA — Compliance & Risk Audit
Zero-compromise auditor. Scores 14 categories against red flag thresholds. CRITICAL flags cost −20 pts each. Score < 50 → NO-GO. Generates questions to the contracting authority and mitigation actions.
Output: compliance table, scoring, red flags, CRAA recommendation
4
TSFA — Technical Scope & Feasibility
Senior site manager persona. Estimates execution time, identifies hidden costs, classifies scope (STANDARD / EXTENDED / COMPLEX), evaluates grid-connection requirements. Uses Code Interpreter for calculations.
Output: scope classification, schedule risk, hidden cost estimates
↓
5
SOA — Strategic Orchestration Agent
Merges CRAA + TSFA outputs. Checks consistency across all upstream agents. Produces the 6-section Executive Insight Report: tender data → formal scoring → technical assessment → final recommendation → strategy → questions to authority.
Output: GO / CONDITIONAL / NO-GO · Executive Insight Report (PDF)

Key technical decisions

Why 5 agents instead of 1?

Each agent operates within its own context, has a defined temperature (0.0 for normalisation, 0.3 for strategy), and enforces its own output contract. A single monolithic prompt cannot hold 14 JSON keys, compliance scoring logic, technical feasibility analysis, and executive synthesis simultaneously without degrading accuracy.

JSON interface contracts

The 14-key FEA→CRAA contract is enforced at runtime. CRAA validates input completeness before proceeding. This makes the pipeline auditable — every fact carries a source tag referencing the exact document, section, and page number it came from.

Security from the start

userdata.get('API_KEY') instead of hardcoding. Client data anonymised before publication. Retry logic with max_retries = 3 and exponential backoff on API calls.
