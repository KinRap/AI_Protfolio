Why this portfolio exists

Technical communication is not separate from AI work — it is the same skill applied to a different output. The ability to structure complex information precisely, understand the audience, and make systems legible is exactly what AI implementation projects need at the business layer.

This portfolio was built during my postgraduate study in Technical Communication (2023–2024) and continues to grow. It's published as a Jekyll site using Just The Docs, maintained on GitHub, and written in Markdown with XML-based structured authoring.

What's in it

Prompt Design Strategies

A structured guide to prompt engineering for technical writers — role assignment, few-shot examples, chain-of-thought, output format control. Written for practitioners, not theorists.

Generative AI for Technical Writing

Practical documentation covering how to integrate GenAI tools into documentation workflows: content generation, review, localisation support, and quality control.

DITA Structured Authoring

Topic-based authoring in DITA XML. Task, concept, and reference topic types. Content reuse with conrefs. Built and validated in Oxygen XML Editor.

MadCap Flare Project

Single-source publishing project producing HTML5 web help and PDF from the same source. Conditional tagging, stylesheet customisation, TOC design.

Simplified English

ASD-STE100 Simplified Technical English — controlled vocabulary, sentence length rules, active voice requirements. Applied to a real maintenance documentation scenario.
